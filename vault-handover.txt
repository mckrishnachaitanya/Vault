FROZEN — Architecture & Decisions
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CORE ARCHITECTURE
-----------------
- 6-digit PIN → PBKDF2 key → AES-256-GCM encrypts entry array → localStorage
- localStorage keys: vault_data, vault_pin, vault_theme, vault_bio,
  vault_bio_cred, vault_lockout, vault_fails, vault_local_updated,
  vault_cloud_sync, vault_cloud_uid, vault_cloud_email, vault_last_sync,
  vault_sync_pending
- IndexedDB "VaultDocsDB" v1: two stores — "meta" (key 'manifest' →
  Crypto.encrypt blob of manifest array) and "files" (key = doc id →
  {salt, iv, ct} object encrypted via Crypto.encryptRaw/decryptRaw).
  Files stored as raw ArrayBuffer — no base64/JSON wrapping, no size
  limit issues on mobile Chrome.
- state{} holds: entries, currentPIN, unlocked, unlockedViaBio,
  stepUpDone, pinBuffer, pinMode, pinTemp, editId, activeFilter,
  currentNav, detailSource, failCount, preLockScreen, formDraft,
  bioGen, bioPending, cloudUser, cloudSyncOn, syncStatus,
  docsStepUpDone, docs[], docViewSource, docViewEntryId
- Auto-lock: 2min idle. Lockout: 5 fails → 30s delay.
- Single file always. No build tooling. No frameworks.

ENTRY SCHEMA
------------
All:      id, type, name, fav, created, updated, customFields[],
          sensitiveOverride, linkedDocIds[]
app:      url, username, password, notes
wifi:     ssid, password, securityType, notes
note:     note
card:     cardHolder, cardNumber(raw digits), expiry(MM/YY), cvv, pin, notes
bank:     bankName, accountNo, ifsc, accountType, netBankingId,
          netBankingPassword, notes
identity: docType(aadhaar/pan/passport/dl/voterid/other), docName,
          docNumber(raw digits for aadhaar), idName, dob(DD/MM/YYYY),
          expiry(DD/MM/YYYY), nationality, issuingState, notes

linkedDocIds is always preserved through form edits (collectFormData
copies from existing entry). Absent on old entries = treated as [].
Max 10 links per entry, enforced in link picker and UI.

DOC MANIFEST SCHEMA
-------------------
Each doc: {id, name, mimeType, size, created, updated, fav,
           category, thumb (data: URI for images only)}
category values: none|identity|medical|property|financial|education|
                 travel|other (see DOC_CATEGORIES array)
Stored encrypted in IndexedDB meta store. In-memory as state.docs[].

BIOMETRIC RULES
---------------
- WebAuthn platform only. Credential ID stored in vault_bio_cred.
- Fresh app open: PIN required once to seed currentPIN in memory.
  Bio auto-triggers on every re-lock after first PIN, via state.bioPending
  + visibilitychange/focus listeners.
- unlockedViaBio=true → step-up PIN required for Card/Bank/Identity
  view, Add/Edit/Delete/Export, form restore after re-auth.
- stepUpDone=true after first step-up (resets on lockVault). Manual
  PIN unlock ALSO sets stepUpDone=true.
- Docs tab: docsStepUpDone controls Docs-specific gate. Passing Docs
  step-up also sets stepUpDone=true (satisfies general step-up).
- Viewing a linked doc from entry detail: step-up fires if
  unlockedViaBio && !docsStepUpDone (same gate as Docs tab).
- "Link a document" button: NO step-up — entry detail step-up already
  happened. Opening the linked doc viewer IS gated as above.

SCREEN RESTORE AFTER RE-AUTH
----------------------------
lockVault() captures preLockScreen: form/detail/docview/fav/settings/docs.
restoreAfterUnlock() restores each appropriately. docview-screen restores
via openDocsTab() + openDocDetail() (step-up re-fires correctly).

SECURITY TIERS
--------------
Sensitive: Card, Bank, Identity (step-up when bio unlock).
Non-sensitive: App, WiFi, Note.
Per-entry override via sensitiveOverride field.
Docs: docsStepUpDone gate (highest tier, PIN once per session).

NUMBER FORMATTING
-----------------
cardNumber and Aadhaar docNumber stored as raw digits.
Formatted on display only: 16 digits → 4-4-4-4, 15 → Amex 4-6-5,
12 (Aadhaar) → 4-4-4. Card expiry stored as MM/YY.

SWIPE-TO-DELETE
---------------
.swipe-delete-bg always opacity:0 in CSS. Never set via class.
Only set to opacity:1 via JS inline style during actual swipe gesture
(snapOpen) and back to 0 on snapClose. This prevents backdrop-filter
on glass cards from bleeding red through overflow:hidden on Android Chrome.

RULED OUT — PERMANENTLY
-----------------------
- Password generator: not in scope.
- Multi-page architecture: single file is intentional for security.
- Storing PIN or decryption key in Firestore: PIN never leaves device.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STORAGE
- DocsDB module: IndexedDB VaultDocsDB v1.
- Manifest encrypted with Crypto.encrypt (small JSON, base64 string).
- File blobs encrypted with Crypto.encryptRaw (ArrayBuffer → {salt,iv,ct}
  stored natively in IndexedDB). No base64 — handles 5MB+ photos.
- Thumbnail for images: 80px JPEG canvas resize, stored as data: URI
  in manifest (fast list display without decrypting file).
- PIN change: DocsDB.reencryptAll(oldPin, newPin) before saveEntries().

DOCS UI
- 5th bottom-nav slot. Lock moved to top-right .lock-top-btn on all screens.
- Docs tab gate: _showStepUpModal() directly (bypasses requirePIN shortcut).
  Sets docsStepUpDone=true AND stepUpDone=true on success.
- File picker: state.importInProgress=true prevents visibilitychange lock.
  cancel event + 'change' both clear it.
- Add: fileToArrayBuffer → DocsDB.saveFile → manifest update → renderDocs().
  Duplicate by filename: "File Already Exists" confirmDelete sheet with
  Replace/Cancel. Replace preserves original fav/created.
- Viewer: images → <img> blob URL. PDFs → "Open PDF" button →
  window.open(blobUrl,'_blank') (new tab, no embed hijack).
  Word/Excel/other → "Open DOCX/XLSX/etc." button same pattern.
  Blob URLs revoked on back/delete/re-open.
- Download: <a download> direct. Share: re-decrypts ArrayBuffer fresh
  (no fetch()) to build File for navigator.share — preserves gesture
  context on Android Chrome.
- Rename: updates topbar, preview card filename, meta card, and silently
  calls renderDocs() so list reflects new name.
- Categories: 8 types (none/identity/medical/property/financial/education/
  travel/other) with icons. Picker in viewer. Badge on list card.
  Filter pills only shown when at least one doc has a real category.
- Search: filters by filename in real time. Resets on tab re-open.
- Swipe-to-delete: doc cards use entry-card class so addSwipeToDelete works.
- Export: decrypts to real files → zip with files/<name> structure.
  manifest.json preserves all fields. User can open files directly from zip.
- Import: reads files/<name> or files/<id>.bin (backward compat).
  Deduplicates by filename (replaces same-name). Preserves category+fav.
  Reports "N added · N replaced · N skipped". JSZip lazy CDN import.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PASSWORD HEALTH
- Stats row changed from 3-card flex row → 2×2 CSS grid.
- 4th stat card: 🛡️ PW Health (id=stat-health-card, purple accent strip).
- healthPanelOpen state var (like catPanelOpen). Panel uses existing
  .cat-inline-panel/.cat-inline-inner CSS — same slide-open pattern as
  Types Used breakdown. Both panels mutually exclusive (opening one closes
  the other).
- checkPasswordHealth() runs on: unlock, every saveEntries() call, nav
  to home. Re-evaluates runHealthCheck() each time.
- runHealthCheck(): checks app/wifi/bank entries for:
  1. Reused passwords (same non-empty pw on 2+ entries)
  2. Weak passwords (passwordScore <= 2, not already flagged as reused)
  3. Missing passwords (no password field set)
- passwordScore(pw): 0-5, same algorithm as form strength bar.
- entryPassword(e): extracts primary pw field per type (app/wifi →
  e.password, bank → e.netBankingPassword).
- Card shows 🛡️ All Good (green) / ⚠️ N Issues (red) / 🔔 N Issues (amber).
- Panel rows are tappable → openDetail(id). Closes panel on nav.
- Card is disabled (opacity .5, no click) if fewer than 2 relevant entries.

  "Clear cache" is safe.

- Always export before PIN change or new deploy.
- After uploading new index.html: clear PWA cache via
  chrome://settings/siteData → github.io → delete.
- Docs export zip is plaintext — store it somewhere safe.
- .vault file (entries backup) is NOT accepted by docs import — correct.
  Docs backup is a .zip file from Settings → Export Documents Backup.
- Share button: confirmed working correctly. Uses a pre-built File
  object (created when the doc viewer loads, before any tap) so
  navigator.share() fires with no async gap — avoids Android losing
  user-gesture context. Falls back to auto-download if share is
  unsupported in a given browser context.
