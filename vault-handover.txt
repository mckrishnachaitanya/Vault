R7 SUMMARY — Multi-Entry Secure Share + ID Card Redesign (v6.0)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

MULTI-SELECT (Home screen only)
- state.selectMode (bool), state.selectedIds (Set) — added to state{}.
  selectedIds intentionally persists across category-filter switches and
  search — this was a deliberate decision, not an oversight.
- ☑️ button in home-header-row (#select-mode-btn) toggles select mode.
  toggleSelectMode(on) — clears selectedIds when turning OFF.
- In select mode, entry-card shows a checkbox (.entry-checkbox) instead of
  fav-star/arrow; clicking the card toggles selection instead of opening
  detail. Swipe actions are disabled while in select mode (addSwipeActions
  is simply not called on that render pass).
- #select-bar (fixed above bottom-nav) appears once selectedIds.size > 0.
  Tapping the count OR the Share button opens the summary sheet — both do
  the same thing by design (summary-before-share was a late addition).
  ✕ button exits select mode entirely.
- openSummarySheet() — bottom sheet listing each selected entry with a
  remove (✕). Removing the last one auto-exits select mode.
  "Continue to Share" closes the sheet and calls openShareModal(entries[]).

SWIPE ACTIONS (Home + Starred entry lists only — NOT Docs)
- New addSwipeActions(container) — parallel implementation to the existing
  addSwipeToDelete(container) (which still powers the Docs list, untouched).
- Reveals 176px (two 88px buttons: Share, then Delete) instead of 88px.
  Layout order left→right = Share, Delete — Delete sits at the far edge by
  design (agreed as "furthest from content" during discussion, not a
  progressive-reveal threshold — both buttons appear together once the
  swipe crosses the open threshold).
- .swiped-actions class (176px margin) — distinct from the old .swiped
  (88px, still used by Docs) so both gestures coexist without CSS clashes.
- Swipe-share button calls openShareModal(entry) directly (single-entry
  quick share, same flow as the in-detail Share button).

SECURE SHARE — MULTI-ENTRY PAYLOAD
- openShareModal() now accepts a single entry OR an array — always
  normalizes internally to _shareTargetEntries[]. Detail-screen's Share
  button and the new swipe-share button pass a single entry (auto-wrapped).
- Share sheet title shows "🔗 Secure Share (N items)" when N > 1.
- Payload shape changed: { entries: [...], expiresAt }. OLDER LINKS with
  the legacy { entry, expiresAt } shape still decrypt fine — svAttemptUnlock
  checks payload.entries first, falls back to [payload.entry].
- Finishing a bulk share (Done button) auto-exits select mode.

ID CARD REDESIGN (Design D) — recipient view
- Old renderShareCard() + share-card-widget/.share-*-card gradient themes
  REMOVED entirely. Replaced by:
  - idCardModel(entry) — maps any entry type to {eyebrow, name, fields[]}.
    fields carry {label, value, emphasis, wide, prose} flags.
  - buildIdCardHTML(entry) — renders the actual .idcard markup: brass left
    edge, eyebrow row (type label + monogram), serif name, 2-col responsive
    field grid. Primary identifier (card/account/document number) gets
    emphasis+wide. Note/app "Notes" fields get prose (wraps, non-mono).
  - Cream/brass palette scoped under .idcard-scope (CSS vars prefixed
    --ic-*) so it never touches the app's own --accent/--bg vars.
  - Cormorant Garamond added to the Google Fonts link (display serif for
    the name); JetBrains Mono (already loaded) reused for field values —
    no extra font weight added.
- CVV/PIN/passwords are shown in full (NOT masked) — the whole point of
  the recipient view is revealing the actual secret. The masked look in
  the design mockups was a placeholder for the static preview only.

RECIPIENT CAROUSEL (multi-entry shares)
- #share-view-card now has fixed chrome (arrows, dots, peek edges, hint,
  count) around a single #share-idcard-slot that gets its innerHTML
  replaced per-slide — the old approach of overwriting the whole card
  container no longer applies.
- initShareCarousel(entries) — called once after successful PIN decrypt.
  Hides all carousel chrome when entries.length === 1 (single-entry shares
  look exactly like before, just with the new card design).
- renderShareCarouselSlide() — updates card content, dot states, arrow
  disabled states (first/last), and peek-edge visibility.
- shareCarouselGo(delta) — bounds-checked step; wired to both arrow clicks
  and a touchstart/touchend swipe listener on #share-carousel (40px
  horizontal threshold, must exceed vertical delta to count as a swipe).

WARNINGS / NOTES FOR NEXT SESSION
- Fav screen (Starred tab) was intentionally left on the OLD single-button
  swipe (addSwipeToDelete) — multi-select/share was only asked for on Home.
  If that's expected to extend to Starred later, renderFavs() needs the
  same treatment as renderEntries().
- Documents are still NOT part of any share payload — entries only. The
  per-entry `includeDocs` toggle discussed for a future session was NOT
  added to the UI yet since it stays dormant until doc-sharing ships;
  revisit idCardModel()/openShareModal() then.
- No test data available in this session to click-test the carousel swipe
  gesture on an actual touch device — logic mirrors the proven
  addSwipeToDelete touch-handling pattern, but worth confirming feel on
  first real use.
